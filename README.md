# CHIP-8 Emulator

A modular, work-in-progress CHIP-8 emulator written in **C99**. This project emulates the classic CHIP-8 virtual machine used in the 1970s/80s for running simple games and programs. The codebase is designed with clean separation of concerns across CPU, memory, display, input, and timer subsystems.

---

## Overview

CHIP-8 is an interpreted programming language developed by Joseph Weisbecker in the mid-1970s. It was originally designed to allow video games to be more easily programmed for early microcomputers. This emulator recreates the CHIP-8 environment so you can run original ROMs.

### Specifications

| Component | Detail |
|-----------|--------|
| **Memory** | 4,096 bytes (0x000 вҖ“ 0xFFF) |
| **Resolution** | 64 Г— 32 monochrome pixels |
| **Registers** | 16 general-purpose 8-bit registers (V0вҖ“VF) |
| **Index Register** | 16-bit register I |
| **Program Counter** | 16-bit |
| **Stack** | 16 levels |
| **Timers** | 60 Hz delay and sound timers |
| **Keypad** | 16-key hex keypad (0вҖ“F) |
| **Fontset** | Built-in 4Г—5 hex digits stored at 0x000вҖ“0x04F |

---

## Features

-  Clean, modular C99 codebase
-  Separated components: CPU, Memory, Display, Input, Timers
-  Loads CHIP-8 ROMs into memory at address `0x200`
-  Built-in CHIP-8 fontset loaded at startup
-  Monochrome 64Г—32 display buffer with XOR sprite drawing
-  Collision detection flag (VF)
-  Delay & sound timer countdown at 60 Hz
-  Test ROM included (`assets/roms/test.ch8`)
-  SDL2-based graphics & input (planned)
-  Full CHIP-8 instruction set (partially implemented)

## Architecture

### CPU

The `Chip8` struct (defined in `cpu.h`) serves as the central state container:

- **`pc`** вҖ” Program Counter, starts at `0x200`
- **`sp`** вҖ” Stack Pointer
- **`V[16]`** вҖ” General-purpose registers (V0вҖ“VF); VF is the flag register
- **`I`** вҖ” 16-bit index register
- **`stack[16]`** вҖ” Subroutine stack
- **`running`** вҖ” Emulator run state

The `emulate_cycle()` function in `cpu.c` implements the fetch-decode-execute loop:
1. **Fetch** вҖ” Read the 16-bit opcode from memory at `pc`
2. **Decode** вҖ” Extract nibbles: `NNN` (address), `X`/`Y` (registers), `N` (nibble)
3. **Execute** вҖ” Dispatch via a switch statement on the top nibble

### Memory

- Total size: **4,096 bytes**
- Programs start at **`0x200`** (512 decimal)
- Addresses `0x000`вҖ“`0x1FF` are reserved for the interpreter/fontset
- `memory.c` initializes RAM to zero and loads the standard 80-byte hex fontset
- `load_rom()` reads a binary ROM file from disk into memory at `0x200`

### Display

- Resolution: **64Г—32** monochrome pixels
- Representation: 2D boolean array `pixels[32][64]`
- `draw_sprite()` renders sprites using **XOR** blending
- Sprites wrap around the screen edges (modulo arithmetic)
- Collision detection: if a pixel is turned off, VF is set to 1
- `update_display()` is currently a stub for SDL rendering

### Input

- 16-button hex keypad mapped to `bool keys[16]`
- `init_input()` resets all key states to false
- `handle_input()` is a stub awaiting SDL event polling

### Timers

- **Delay Timer** вҖ” Used for game timing; decremented at 60 Hz when non-zero
- **Sound Timer** вҖ” When non-zero, a beep plays; also decremented at 60 Hz
- `update_timers()` decrements both timers each cycle (caller should throttle to ~60 Hz)

## Building

### Prerequisites

- A C compiler (`gcc`, `clang`, or MSVC)
- `make` (for using the provided Makefile)
- *(Optional)* SDL2 development libraries вҖ” for future graphics/input support

### Compile

From the project root directory:

```bash
make
```

This will:
1. Create the `build/` directory if it doesn't exist
2. Compile all `.c` files in `src/` to object files in `build/`
3. Link them into the executable `chip8`

To clean build artifacts:

```bash
make clean
```

> **Note:** The Makefile currently does not link SDL2 (`LIBS` is empty). Once SDL2 is integrated, uncomment the `LIBS = -lSDL2 -lSDL2main` line.

## Implemented CHIP-8 Instructions

| Opcode | Mnemonic | Description |
|--------|----------|-------------|
| `00E0` | `CLS` | Clear the display |
| `00EE` | `RET` | Return from subroutine |
| `1NNN` | `JP addr` | Jump to address NNN |
| `2NNN` | `CALL addr` | Call subroutine at NNN |
| `3XKK` | `SE Vx, byte` | Skip next instruction if Vx == KK |
| `6XKK` | `LD Vx, byte` | Load byte KK into Vx |
| `ANNN` | `LD I, addr` | Load address NNN into index register I |
| `DXYN` | `DRW Vx, Vy, nibble` | Draw sprite at (Vx, Vy) with height N |

### Opcode Breakdown

| Bits | Meaning |
|------|---------|
| `NNN` | 12-bit address |
| `X` / `Y` | 4-bit register identifiers |
| `N` | 4-bit constant (nibble) |
| `KK` / `byte` | 8-bit constant |
